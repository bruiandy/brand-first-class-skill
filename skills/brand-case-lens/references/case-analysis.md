# Case Analysis Playbook

## Good Inputs

- A public brand case
- A launch page or product screenshot
- A social post idea
- A campaign summary
- A founder interview
- A short article draft

## How To Choose Lenses

### Start With 1-2 Lenses

Do not use all five lenses by default.

- If the case is about a new brand or a major shift, start with resource + operating.
- If the case is about positioning, naming, slogan, or storytelling, start with brand + product.
- If the case is about a channel move, content move, or growth result, start with closed-loop + operating.
- If the case is about user love, repeat purchase, or differentiation, start with product + brand.

## What A Good Analysis Should Do

- Identify the non-obvious thing that actually matters.
- Explain what conditions made the move work.
- Warn against copying the surface move without the underlying fit.
- Keep the tone grounded and specific.

## What A Good Analysis Should Avoid

- Turning every case into proof that the framework explains everything.
- Praising tactics without asking what resource or channel condition enabled them.
- Acting certain when the case evidence is thin.

## Output Patterns

### 1. Short Post

- One sharp opening judgment
- Two supporting points
- One sentence on what should not be copied blindly

### 2. Article Angle

- 核心观点
- 为什么这个案例值得看
- 按 1-2 个透镜拆解
- 普通品牌最容易误学什么
- 如果换成另一个阶段，这个方法是否还成立

### 3. Comparison

Compare two cases by asking:

- Which one had the stronger real resource
- Which one had the clearer operating fit
- Which one actually formed a repeatable loop
- Which lesson is portable and which is not

## Required Ending Moves

Close with both of these:

- `这个判断成立的前提是……`
- `目前还不能下结论的部分是……`

That keeps the framework useful without turning it into dogma.
