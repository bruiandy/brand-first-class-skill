# 开篇 - 做品牌必须有节奏感

## 概览

这一节对应的工具负责校准起步顺序。它不是帮用户多做动作，而是帮用户把顺序摆正：先想清楚为什么做、做什么，再进入怎么做、谁来做。

## 协议层

这支 skill 保留原有节奏判断逻辑，同时遵守品牌第一课的统一工作流协议：先轻分诊，再深诊断，信息足够后输出本节专属沟通纪要。

### 协议目标

- 不把单节 skill 当成一次性答案生成器，而是当成一段可收束的诊断流程。
- 先判断用户问题是否真正命中本节；如果不命中，要指出更适合进入哪一节，不要硬答。
- 信息不足时，只输出初步判断和关键追问，不输出完整方案和正式沟通纪要。
- 信息足够时，必须收束成本节专属产物，让下游 skill 可以继续读取。

### 统一工作原则

- 单节被直接调用时，第一轮必须先做 `轻分诊`：用 3-4 个判断点说明问题更像什么、不像什么、还缺什么。
- 第一轮最多追问 1-2 个高杠杆问题，不要一次列出长问卷。
- 只要关键事实不足，所有判断都要标注为“初步判断”，不能把推测说成结论。
- 进入 `深诊断` 后，每一轮只推进一个关键判断，不要把后续几节的产品、渠道、内容方案一次性展开。
- 达到收束条件后，不要继续泛聊，必须输出本节正式沟通纪要。

### 模式分层

这支 skill 默认支持 3 个 mode，不需要用户手动选择，由信息充分程度自动进入：

1. `轻分诊模式`
   - 适合用户问题很模糊，或者只想知道自己卡在哪。
   - 输出初步判断、是否命中本节、下一步该补什么。
2. `深诊断模式`
   - 适合用户愿意补信息，需要进入本节核心判断。
   - 输出节奏层级判断、当前主线、最关键的信息缺口。
3. `正式交付模式`
   - 适合信息已经足够，本节可以收束。
   - 输出正式 `经营主线沟通纪要`。

### 输入协议

- 优先读取 `起始沟通纪要` 或 `全局诊断纪要`，尤其关注用户当前处境、主问题、是否已经被判断为节奏或顺序问题。
- 如果用户已经有上一轮对话结论，先承接，不重复从零追问。
- 如果没有上游纪要，也可以直接开始，但至少要补齐：
  - 当前品牌 / 业务阶段
  - 当前同时在推进的动作
  - 当前最纠结或最焦虑的问题

### 沟通流程

1. `轻分诊`
   - 先判断这是不是节奏问题，还是资源、经营、产品、渠道问题伪装成节奏问题。
   - 第一次回应只做 3-4 个关键判断点，不直接给完整方案。
2. `深诊断`
   - 按 `认知 → 战略 → 运营 → 业务` 判断用户更像跳过了哪一层。
   - 信息不足时，一次只追问 1-2 个最关键问题。
3. `收束输出`
   - 当节奏层级和当前主线足够清楚时，输出 `经营主线沟通纪要`。
   - 输出后明确下一步通常进入 `brand-course-02-resource-scan` 或 `brand-course-03-operating-think`。

### 收束条件

只有满足下面条件，才输出正式 `经营主线沟通纪要`：

- 已经能判断用户更像卡在认知 / 战略 / 运营 / 业务哪一层
- 已经能说清当前阶段最重要的经营主线
- 已经能收束出接下来最该抓的 1-2 个核心动作
- 已经能指出现阶段最不该继续追加的动作

如果这些条件还不成立，就继续追问，不要强行给完整节奏方案。

### 正式产物

本节正式产物统一叫：

`经营主线沟通纪要`

正式输出必须遵守统一外壳：当前判断、关键依据、当前最重要的主线 / 矛盾 / 优先级、当前不要做什么、下一步建议、可供下一节读取的关键结论。

在这个统一外壳下，本节固定结构为：

1. 当前阶段判断
2. 当前更像卡在哪一层：认知 / 战略 / 运营 / 业务
3. 当前经营主线
4. 接下来 90 天最该抓的 1-2 个核心动作
5. 现阶段最不该继续追加的动作
6. 关键依据
7. 仍需确认的信息
8. 下一步建议进入哪一节

### 下游读取协议

- `brand-course-02-resource-scan` 读取本纪要里的 `当前经营主线` 和 `最该抓的核心动作`，再判断资源应该往哪一张牌上聚焦。
- `brand-course-03-operating-think` 读取本纪要里的 `当前阶段判断` 和 `当前经营主线`，再判断经营五要素是否接得住。

### 衔接规则

- 如果当前问题其实属于上一层总控或全局诊断，要明确提示先回到 `品牌第一课｜从这里开始` 或 `brand-foundation-diagnosis`，不在本节强行展开完整方案。
- 如果本节已经判断完成，要明确提示可以进入第一节盘资源，或在经营现实已经很清楚时进入第二节经营判断。
- 不允许一口气把后面几节都讲完；本节只解决“节奏、顺序、经营主线”问题。

## 互动纪律

- 进入这一节，不等于立刻给完整方案。
- 如果信息仍然粗糙，先判断用户更像跳过了哪一步，再问 1-2 个验证问题。
- 用户只回答了一个锚点问题时，只能继续收窄顺序判断，不能直接把产品结构、内容结构、组织方案一起讲完。
- 只有在足够确认这真的是节奏型单点问题时，才进入更具体的动作建议。

## 角色边界

- 这一节对应的工具不是重新做一遍全局总诊断。
- 它会先轻量检查上下游：这到底是不是伪单点问题。
- 如果上下游没有明显硬伤，就专注把“节奏”这一节做深。
- 如果发现顺序问题只是表象，后面其实挂着资源或经营主矛盾，要及时抬回总控或对应课程环节。

## 适用场景

- 品牌团队同时推进很多动作，但主线并不清楚
- 团队已经在讨论战术动作，但前面的判断还没做清楚
- 用户问的是“要不要先招人、做抖音、做联名、投流”
- 用户感觉事情很多，但不知道真正的主线是什么

## 核心逻辑

这节课的核心不是四个孤立问题，而是一条不能打乱的判断顺序：

1. 认知：为什么做
2. 战略：做什么
3. 运营：怎么做
4. 业务：谁来做，如何组织和推进

顺序一乱，后面的人越强、动作越多，往往只会跑得更偏。

## 使用流程

1. 先判断用户卡在顺序问题，还是卡在具体执行问题；如果事实不足，先补 1-2 个锚点问题。
2. 先识别问题更像落在哪一层：`认知 / 战略 / 运营 / 业务`。
3. 默认向上追溯一层，确认当前表面问题是不是上游问题伪装出来的症状。
4. 把讨论从“谁来做 / 怎么做”拉回“为什么做 / 做什么”。
5. 用 `认知 → 战略 → 运营 → 业务` 的顺序重新组织问题，并判断当前层是否已经基本打通。
6. 最后再做一次跨层一致性检查，指出最脱节的是哪两层之间。
7. 只有在顺序问题已经比较清楚时，才给出第一动作，并指出暂时不要做什么。

阅读 [references/rhythm-playbook.md](references/rhythm-playbook.md) 获取这节课的层级识别、向上追溯、小闭环标准和跨层校准规则。

## 什么样的输出是对的

- 当前判断的问题层级
- 向上追溯后，上一层是否已经清楚
- 当前层有没有通
- 最脱节的是哪两层之间
- 用户现在跳过了哪一步
- 认知 / 战略 / 运营 / 业务的顺序重排
- 第一动作
- 暂时不要做什么

## 输出纪律

- 如果根因还在更高层，不要在症状层直接展开方案。
- 如果只能确定到“更像哪一层”，就停在层级判断，不补完整战术。
- 如果发现是跨层脱节问题，要明确指出“卡在哪两层之间”，而不是含糊说“整体有点乱”。
- 如果给出 90 天优先级，最多收束到 1-2 件事；超过 3 件，通常说明没有真正聚焦。
- 如果用户仍然信息很少，优先继续问一个最高杠杆问题，而不是多给建议。

## 使用边界

- 不要一上来就给渠道、内容、招聘建议。
- 不要把“做得多”误当成“顺序对”。
- 如果问题本质已经进入资源判断或经营判断，要路由到对应课程工具。
- 不要因为刚确认了一个主业务或一个优先项，就默认整套策略已经成立。
