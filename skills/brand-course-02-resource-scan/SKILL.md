# 第一节 - 盘资源

## 概览

这一节对应的工具用来做资源扫描。它不是帮用户把所有资源都讲成优势，而是帮用户看清：在当前阶段、当前战场和当前团队现实下，到底哪一张牌的相对胜算最高，哪一些方向现在不该做。

## 协议层

这支 skill 保留原有资源扫描和相对优势判断逻辑，同时遵守品牌第一课的统一工作流协议：先轻分诊，再深诊断，信息足够后输出本节专属沟通纪要。

### 协议目标

- 不把单节 skill 当成一次性答案生成器，而是当成一段可收束的诊断流程。
- 先判断用户问题是否真正命中本节；如果不命中，要指出更适合进入哪一节，不要硬答。
- 信息不足时，只输出初步判断和关键追问，不输出完整资源配置方案和正式沟通纪要。
- 信息足够时，必须收束成本节专属产物，让下游 skill 可以继续读取。

### 统一工作原则

- 单节被直接调用时，第一轮必须先做 `轻分诊`：用 3-4 个判断点说明这是不是资源聚焦问题、是不是伪优势、还缺什么事实。
- 第一轮最多追问 1-2 个高杠杆问题，不要一次列出长问卷。
- 只要关键事实不足，所有判断都要标注为“初步判断”，不能把某个资源点直接定成核心优势。
- 进入 `深诊断` 后，每一轮只推进一个关键判断，按 `理性盘点 → 聚焦优势 → 差异化配置` 收束。
- 达到收束条件后，不要继续泛聊，必须输出本节正式沟通纪要。

### 模式分层

这支 skill 默认支持 3 个 mode，不需要用户手动选择，由信息充分程度自动进入：

1. `轻分诊模式`
   - 适合用户资源很多但说不清重点，或者只想判断是不是资源问题。
   - 输出初步判断、是否命中本节、下一步该补什么。
2. `深诊断模式`
   - 适合用户愿意补信息，需要进入资源扫描和相对优势判断。
   - 输出资源全景、伪优势识别、当前最值得验证的一张牌。
3. `正式交付模式`
   - 适合信息已经足够，本节可以收束。
   - 输出正式 `核心优势沟通纪要`。

### 输入协议

- 优先读取 `经营主线沟通纪要`，尤其关注当前阶段、当前经营主线、接下来 90 天最该抓的核心动作。
- 如果没有上游纪要，也可以直接开始，但要先补齐：
  - 当前品牌 / 业务阶段
  - 当前已经拥有的关键资源
  - 当前最想推进的方向
  - 当前最不确定该押哪张牌
- 如果用户只给了一个资源点，不要立刻把它定义成优势，先验证。

### 沟通流程

1. `轻分诊`
   - 先判断这是不是资源聚焦问题，还是节奏、经营、产品问题伪装成资源问题。
   - 第一次回应只做初步判断，不直接给完整配置方案。
2. `深诊断`
   - 按 `理性盘点 → 聚焦优势 → 差异化配置` 推进。
   - 用创始人 / 团队 / 运营基础三大维度建立资源全景。
   - 用匹配、闭环、可持续和相对胜算验证这是不是相对优势。
3. `收束输出`
   - 当最值得押的一张牌已经相对清楚时，输出 `核心优势沟通纪要`。
   - 输出后明确下一步通常进入 `brand-course-03-operating-think`，验证这张牌放进经营系统里是否成立。

### 收束条件

只有满足下面条件，才输出正式 `核心优势沟通纪要`：

- 已经能收束出当前最值得押的一张牌
- 已经能解释它为什么是相对优势，而不是绝对自嗨
- 已经能区分哪些资源只是表面资源或伪优势
- 已经能说明这张牌更适合落到产品、渠道、信任、组织还是分发上
- 已经能指出现阶段不该平均发力的方向

如果这些条件还不成立，就继续追问，不要强行给品牌、产品、营销完整方案。

### 正式产物

本节正式产物统一叫：

`核心优势沟通纪要`

正式输出必须遵守统一外壳：当前判断、关键依据、当前最重要的主线 / 矛盾 / 优先级、当前不要做什么、下一步建议、可供下一节读取的关键结论。

在这个统一外壳下，本节固定结构为：

1. 当前资源全景
2. 当前最值得押的一张牌
3. 为什么它是相对优势
4. 哪些资源只是表面资源 / 伪优势
5. 这张牌更适合落到哪里：产品 / 渠道 / 信任 / 组织 / 分发
6. 现阶段不该平均发力的方向
7. 关键依据
8. 下一节经营思维要重点验证什么

### 下游读取协议

- `brand-course-03-operating-think` 读取本纪要里的 `最值得押的一张牌`、`相对优势依据` 和 `落点方向`，再判断这张牌放进品类 / 渠道 / 团队 / 品牌 / 毛利五维经营系统里是否成立。
- `brand-course-04-brand-ultimate-question` 可以读取本纪要里的 `相对优势`，判断品牌底层命题是否和这张牌一致。

### 衔接规则

- 如果当前问题其实属于第一节节奏问题，要明确提示先回第一节补经营主线，不在本节强行判断优势。
- 如果本节已经判断完成，要明确提示可以进入第二节经营判断，验证这张牌放进真实经营系统后是否成立。
- 不允许一口气把后面的品牌、产品、渠道、内容方案都讲完；本节只解决“资源、优势、聚焦”问题。

## 互动纪律

- 进入这一节，不等于马上替用户决定完整业务结构。
- 如果资源信息还不够，先问 1-2 个最能暴露长板的验证问题。
- 用户刚说出一个资源点时，只能先判断“这可能是一张牌”，不能立刻把后面的品牌、产品、营销方案全部展开。
- 只有在这张牌已经经过阶段匹配、闭环关键、可持续和相对胜算的基本验证后，才进入更具体的聚焦建议。

## 角色边界

- 这一节对应的工具负责把“优势”看清，不负责重做整套总诊断。
- 它要先轻量检查这个资源问题是否被节奏问题、经营现实或战场选择扭曲。
- 如果上下游没有明显硬伤，就把长板、伪长板、相对胜算和集中配置逻辑做深。
- 如果发现真正卡点不在资源，而在经营或产品闭环，要明确指出并转回对应模块。

## 适用场景

- 用户想知道品牌应该聚焦什么
- 用户在问自己的不公平优势是什么
- 用户手里资源很多，但不知道哪一个是真长板
- 用户想判断现有资源能不能支撑接下来的品牌动作
- 用户已经做过第一节节奏判断，现在想知道该用哪一张牌往下推

## 核心问题

什么样的资源，既匹配当前阶段，又能支撑商业闭环，还能持续放大，并且在你选择的战场上，相对竞争对手有更高胜算？

## 资源三大维度

1. 创始人
2. 团队
3. 运营基础

这三个维度是顶层分类。具体细项可以在每个维度下面展开，但先不要一开始就把资源拆得过细，免得失去判断主线。

## 使用流程

1. 先确认用户有没有做过第一节；如果做过，先接住“当前最该推进的是哪一层”；如果没做过，先补 1-2 个问题判断当前阶段。
2. 再用三大维度建立资源全景；如果全景还不够，先补 1-2 个关键事实。
3. 找到最可能成为不公平优势的候选牌，同时识别哪些只是表面资源或伪长板。
4. 用四问验证：是否匹配当前阶段、是否是闭环关键、是否可持续、在当前战场上相对竞争对手是否更有胜算。
5. 判断这张牌更适合推进哪一层：认知、战略、运营还是业务，不要在错误层面放大资源。
6. 最后做一次跨层一致性检查，确认这张牌和节奏、战场、团队现实是不是同一条线。
7. 只有在判断已经比较清楚时，才给出聚焦建议，并明确哪些方向现在应该收缩或暂缓。

阅读 [references/resource-map.md](references/resource-map.md) 获取资源识别、伪优势识别、真优势验证、小闭环标准和跨层校准规则。

## 什么样的输出是对的

- 资源概览
- 当前最该推进的层级
- 候选优势里哪一个最可能是真优势
- 哪些更像伪优势或表面资源
- 为什么这张牌的相对胜算最高
- 应该聚焦的长板
- 该收缩或暂缓的方向
- 下一步还要补的关键信息

## 输出纪律

- 不要把“资源多”误写成“优势强”。
- 不要把“我最喜欢这张牌”误写成“这张牌最该押”。
- 如果用户还没确认当前阶段，就不要直接下聚焦结论。
- 如果这张牌离闭环很远，就不要硬夸成主优势。
- 如果相对胜算还没有看清，只能停在候选牌判断，不能直接展开完整打法。

## 使用边界

- 不要把“资源多”误当成“优势强”。
- 不要把短板硬解释成长板。
- 不要把“绝对最强”误当成“当前最值得押”。
- 如果经营现实本身已经卡死路径，要明确转到经营判断。
- 不要因为刚找到一个像长板的点，就默认它已经足以支撑完整品牌打法。
