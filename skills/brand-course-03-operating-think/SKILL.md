# 第二节 - 建立业务经营思维

## 概览

这支 lesson skill 负责经营判断。核心不是“品牌怎么讲得更好”，而是先回答：这个品类、这组渠道、这支团队、这套毛利结构，究竟支不支撑你想做的增长路径。

## 互动纪律

- 进入这一节，不等于立刻给出完整经营路径。
- 如果品类、渠道、团队、毛利还没有基本事实，就先补 1-2 个验证问题。
- 用户刚确认了一个渠道或一个主产品时，只能帮助收窄经营判断，不能立刻把后面的闭环、内容、组织方案一起讲完。
- 只有在经营现实已经足够清楚，或问题明显是单点经营问题时，才进入更具体的路径建议。

## 角色边界

- 这支 lesson skill 负责把经营现实讲清楚，不负责重做整套系统判断。
- 它会先轻量检查：这个经营问题到底是不是被资源、产品价值或闭环表现误导出来的。
- 如果经营确实是主矛盾，就在这一节里深挖品类、渠道、团队、毛利。
- 如果经营只是表象，后面真正主矛盾在别处，要明确指出并切回对应模块。

## 适用场景

- 用户在问哪个渠道更合理
- 用户在问该走品牌导向、渠道导向、产品导向还是内容导向
- 用户需要判断毛利和类目限制
- 用户想要的是一条可行的经营路径，而不是一句口号

## 核心逻辑

品牌是果，经营是因。  
经营判断先看四个顶层元素：

1. 品类
2. 渠道
3. 团队
4. 毛利

这四个元素决定你今天能不能做、该怎么做，以及品牌打法应该从哪里长出来。

## 使用流程

1. 先判断品类属性；如果还不清楚，先补关键事实。
2. 再判断渠道逻辑。
3. 再看团队配置是否匹配。
4. 再看毛利结构能否支撑动作。
5. 如果经营判断还在早期，就先指出主要矛盾和下一步最该补的信息。
6. 最后给出可行的经营路径，而不是抽象建议。

阅读 [references/operating-matrix.md](references/operating-matrix.md) 获取四个顶层元素的展开和输出模板。

## 什么样的输出是对的

- 品类判断
- 渠道判断
- 团队判断
- 毛利判断
- 可行的经营方向
- 现在不适合做什么

## 使用边界

- 不要因为品牌表达动人，就忽略经营现实。
- 不要因为某个渠道火，就默认它适合这个品牌。
- 不要把经营问题假装成品牌问题。
- 不要因为刚确认一个经营锚点，就把整套解决方案一次说完。
