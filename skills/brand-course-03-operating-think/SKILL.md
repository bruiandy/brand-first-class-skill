# 第二节 - 建立业务经营思维

## 概览

这一节对应的工具负责经营判断。核心不是“品牌怎么讲得更好”，而是先回答：这个品类、这组渠道、这支团队、这套品牌打法、这套毛利结构，究竟支不支撑你想做的增长路径，以及第二节里找到的那张牌，放进真实经营系统后还成不成立。

## 协议层

这支 skill 保留原有五维经营联动判断逻辑，同时遵守品牌第一课的统一工作流协议：先轻分诊，再深诊断，信息足够后输出本节专属沟通纪要。

### 协议目标

- 不把单节 skill 当成一次性答案生成器，而是当成一段可收束的诊断流程。
- 先判断用户问题是否真正命中本节；如果不命中，要指出更适合进入哪一节，不要硬答。
- 信息不足时，只输出初步判断和关键追问，不输出完整经营路径和正式沟通纪要。
- 信息足够时，必须收束成本节专属产物，让下游 skill 可以继续读取。

### 统一工作原则

- 单节被直接调用时，第一轮必须先做 `轻分诊`：用 3-4 个判断点说明这是不是经营系统问题、哪个要素最可疑、还缺什么事实。
- 第一轮最多追问 1-2 个高杠杆问题，不要一次列出长问卷。
- 只要品类、渠道、团队、品牌、毛利里的关键事实不足，所有判断都要标注为“初步判断”。
- 进入 `深诊断` 后，每一轮只推进一个关键判断，重点检查五个要素如何互相限制、互相嵌套、互相拖拽。
- 达到收束条件后，不要继续泛聊，必须输出本节正式沟通纪要。

### 模式分层

这支 skill 默认支持 3 个 mode，不需要用户手动选择，由信息充分程度自动进入：

1. `轻分诊模式`
   - 适合用户只抛出一个渠道、品类、毛利或打法问题。
   - 输出初步判断、是否命中本节、最需要补的经营事实。
2. `深诊断模式`
   - 适合用户愿意补信息，需要进入五维经营联动判断。
   - 输出五维之间的主要矛盾、第二节那张牌是否接得住、当前最脱节的环节。
3. `正式交付模式`
   - 适合信息已经足够，本节可以收束。
   - 输出正式 `经营联动沟通纪要`。

### 输入协议

- 优先读取 `核心优势沟通纪要`，尤其关注第二节判断出的那张牌、相对优势依据、落点方向。
- 也可以读取 `经营主线沟通纪要` 或 `全局诊断纪要`，确认当前阶段和经营主线。
- 如果没有上游纪要，也可以直接开始，但要先补齐：
  - 当前品类和主产品
  - 当前主渠道或最想进入的渠道
  - 当前团队最强和最弱的能力
  - 当前品牌打法或内容打法
  - 当前毛利结构或大致价格带
- 如果用户只提供一个渠道、一个产品或一个毛利数字，不要立刻判断整条路成立，先验证五维之间是否互相接得住。

### 沟通流程

1. `轻分诊`
   - 先判断这是不是经营系统问题，还是资源、品牌表达、产品价值或营销闭环问题伪装成经营问题。
   - 第一次回应只做 3-4 个关键判断点，不直接给完整经营方案。
2. `深诊断`
   - 围绕 `品类 → 渠道 → 团队 → 品牌 → 毛利` 做联动判断。
   - 重点不是逐项打分，而是看五个要素之间怎么互相限制、互相嵌套、互相拖拽。
   - 必须回到第二节那张牌，判断它放进真实经营系统后还成不成立。
3. `收束输出`
   - 当经营主矛盾和五维联动关系足够清楚时，输出 `经营联动沟通纪要`。
   - 输出后明确下一步通常进入 `brand-course-04-brand-ultimate-question`，把经营现实往品牌底层命题上承接。

### 收束条件

只有满足下面条件，才输出正式 `经营联动沟通纪要`：

- 已经能判断品类、渠道、团队、品牌、毛利五个要素的基本状态
- 已经能判断第二节那张牌放进这套经营系统后是否成立
- 已经能说清当前更适合哪种经营导向：产品导向、渠道导向、内容导向、品牌导向，还是先补组织能力
- 已经能指出当前最脱节的是哪两个要素
- 已经能判断先修哪个、先不要动哪个

如果这些条件还不成立，就继续追问，不要强行给完整经营路径。

### 正式产物

本节正式产物统一叫：

`经营联动沟通纪要`

正式输出必须遵守统一外壳：当前判断、关键依据、当前最重要的主线 / 矛盾 / 优先级、当前不要做什么、下一步建议、可供下一节读取的关键结论。

在这个统一外壳下，本节固定结构为：

1. 当前经营现实概览
2. 五维判断：品类 / 渠道 / 团队 / 品牌 / 毛利
3. 第二节那张牌是否还能成立
4. 当前更适合的经营导向
5. 当前最脱节的两个要素
6. 应该先修哪个
7. 现阶段先不要动哪个
8. 当前更可行的经营路径
9. 关键依据
10. 下一节品牌终极之问要重点验证什么

### 下游读取协议

- `brand-course-04-brand-ultimate-question` 读取本纪要里的 `经营导向`、`更可行的经营路径` 和 `下一节要重点验证什么`，判断 VMV 是否和经营现实断裂。
- `brand-course-05-brand-narrative-system` 可以读取本纪要里的 `品牌打法` 和 `经营路径`，判断叙事体系是否接得住真实业务。
- `brand-course-07-channel-product-loop` 可以读取本纪要里的 `渠道判断`、`团队判断` 和 `毛利判断`，继续做产品和渠道适配。

### 衔接规则

- 如果当前问题其实属于第二节资源优势问题，要明确提示先回第二节补“最值得押的一张牌”，不在本节强行做经营路径。
- 如果本节已经判断完成，要明确提示可以进入第四节品牌终极之问，检查这套经营现实能不能长出成立的品牌底层命题。
- 不允许一口气把 VMV、产品卖点、渠道活动和营销闭环都讲完；本节只解决“经营五维是否互相接得住”问题。

## 互动纪律

- 进入这一节，不等于立刻给出完整经营路径。
- 如果品类、渠道、团队、毛利还没有基本事实，就先补 1-2 个验证问题。
- 用户刚确认了一个渠道或一个主产品时，只能帮助收窄经营判断，不能立刻把后面的闭环、内容、组织方案一起讲完。
- 只有在经营现实已经足够清楚，第二节那张牌也已经过了基本校验，才进入更具体的路径建议。

## 角色边界

- 这一节对应的工具负责把经营现实讲清楚，不负责重做整套系统判断。
- 它会先轻量检查：这个经营问题到底是不是被资源判断、产品价值或闭环表现误导出来的。
- 如果经营确实是主矛盾，就在这一节里深挖品类、渠道、团队、品牌、毛利之间的联动关系。
- 如果经营只是表象，后面真正主矛盾在别处，要明确指出并切回对应模块。

## 适用场景

- 用户在问哪个渠道更合理
- 用户在问该走品牌导向、渠道导向、产品导向还是内容导向
- 用户需要判断毛利和类目限制
- 用户想要的是一条可行的经营路径，而不是一句口号
- 用户已经在第二节找到一张想押的牌，现在想判断经营现实接不接得住

## 核心逻辑

品牌是果，经营是因。  
经营判断不是把五个词逐项检查完，而是要看这些要素能不能拧成一条成立的路径。

先看五个顶层元素：

1. 品类
2. 渠道
3. 团队
4. 品牌
5. 毛利

这五个要素不是并列罗列，而是互相限制、互相嵌套。它们共同决定你今天能不能做、该怎么做，以及第二节那张牌到底能不能放进去。

## 使用流程

1. 先确认用户在第二节里找到的那张牌是什么；如果没有，就先补一句“当前最想押哪一张牌”。
2. 先判断品类属性；如果还不清楚，先补关键事实。
3. 再判断渠道逻辑，看主渠道应该怎么选。
4. 再看团队配置是否匹配，看团队能不能执行这条路。
5. 再看品牌打法是否和渠道、产品、团队匹配。
6. 最后看毛利结构能否支撑动作，看这张牌值不值得持续投入。
7. 做一次跨要素核对，指出当前最脱节的是哪两个点、先修哪个、哪个先不要动。
8. 如果经营判断还在早期，就先指出主要矛盾和下一步最该补的信息；如果已经足够清楚，再给出可行路径，而不是抽象建议。

阅读 [references/operating-matrix.md](references/operating-matrix.md) 获取经营联动矩阵、互相约束规则、小闭环标准和跨要素核对规则。

## 什么样的输出是对的

- 品类判断
- 渠道判断
- 团队判断
- 品牌判断
- 毛利判断
- 第二节那张牌放进这套经营系统后还成不成立
- 当前更适合哪种经营导向
- 当前最脱节的是哪两个要素之间
- 应该先修哪个，先不要动哪个
- 可行的经营方向
- 现在不适合做什么

## 输出纪律

- 不要把经营判断写成四项清单点评，要明确这些要素之间怎么互相牵制。
- 不要因为某个单点成立，就默认整条经营路径成立。
- 如果第二节那张牌放不进去，要明确告诉用户“卡在哪”，而不是含糊说经营现实有问题。
- 如果毛利、渠道或团队明显接不住，就不要继续往复杂打法上推。
- 如果还不能确认主矛盾，只能停在联动关系判断，不能直接展开整套品牌打法。

## 使用边界

- 不要因为品牌表达动人，就忽略经营现实。
- 不要因为某个渠道火，就默认它适合这个品牌。
- 不要把经营问题假装成品牌问题。
- 不要把品类、渠道、团队、毛利拆开看，好像彼此互不影响。
- 不要因为刚确认一个经营锚点，就把整套解决方案一次说完。
