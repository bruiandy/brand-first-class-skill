# 第四节 - 构建品牌叙事体系

## 概览

这支 lesson skill 负责把底层品牌命题落到完整的话语体系。它不是帮用户堆一堆文案，而是把同一套核心价值，串成定位、命名、Slogan、视觉符号和关键触点，让品牌说的是同一件事。

## 互动纪律

- 进入这一节，不等于马上出整套定位、命名、Slogan。
- 如果底层命题、用户对象、核心价值还不清楚，先补 1-2 个验证问题。
- 用户刚确认一个方向时，只能先判断叙事主线，不要立刻把整套表达层全部写满。
- 只有在底层命题足够清楚时，才进入完整表达草案。

## 角色边界

- 这支 lesson skill 负责把品牌表达系统做深，不负责重做全局总诊断。
- 它要先轻量检查：表达问题是不是因为底层命题、产品价值或渠道逻辑没站稳。
- 如果表达层确实是主突破口，就在这一节做定位、命名、Slogan、视觉与触点。
- 如果表达只是后果，不是主因，要明确指出并切回对应模块。

## 适用场景

- 用户想做定位、命名、Slogan
- 用户觉得品牌表达很散，内容、包装、页面、销售说法不一致
- 团队已经有一些品牌素材，但没有形成一套完整语言系统
- 用户的 VMV 已经初步清楚，想继续往外落地

## 核心逻辑

品牌表达不是一个个零部件拼起来的，而是同一套核心价值向外展开：

1. 定位：你是谁
2. 命名：你叫什么
3. Slogan：你要让用户最快记住什么
4. 视觉符号：你用什么被识别
5. 关键触点：你在哪些地方反复兑现同一件事

如果这些部分各说各话，品牌就很难形成稳定认知。

## 使用流程

1. 先确认用户的初心、服务对象和价值立场是否已经清楚；不清楚就先补事实。
2. 提炼一句可以承载品牌的核心表达。
3. 如果当前只适合做到主线判断，就先停在这里，不急着展开全部表达层。
4. 再把它展开到定位、命名、Slogan。
5. 再看视觉符号和关键触点怎么围绕同一价值服务。
6. 输出一版可继续打磨的话语体系草案。

阅读 [references/brand-narrative-playbook.md](references/brand-narrative-playbook.md) 获取表达层级、常见错法和输出模板。

## 什么样的输出是对的

- 核心表达主线
- 定位句草案
- 命名方向
- Slogan 方向
- 视觉符号方向
- 关键触点建议

## 使用边界

- 不要在底层命题没清楚时硬做表达。
- 不要把命名、Slogan、视觉当成彼此无关的工作。
- 如果用户还在 0 到 1 阶段、尚无任何品牌素材，这一节的输出应是方向草案，不是表达缺口诊断。
- 如果问题已经回到产品价值和用户任务，要继续路由到产品价值塑造。
- 不要因为刚确认一个主方向，就把整套叙事系统一次性补齐。
