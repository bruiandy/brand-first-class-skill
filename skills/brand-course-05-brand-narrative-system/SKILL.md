# 第四节 - 构建品牌叙事体系

## 概览

这支 lesson skill 负责把底层品牌命题落到完整的话语体系。它不是帮用户堆一堆文案，而是把同一套核心价值，串成定位、命名、Slogan、视觉符号和关键触点，让品牌说的是同一件事。

## 适用场景

- 用户想做定位、命名、Slogan
- 用户觉得品牌表达很散，内容、包装、页面、销售说法不一致
- 团队已经有一些品牌素材，但没有形成一套完整语言系统
- 用户的 VMV 已经初步清楚，想继续往外落地

## 核心逻辑

品牌表达不是一个个零部件拼起来的，而是同一套核心价值向外展开：

1. 定位：你是谁
2. 命名：你叫什么
3. Slogan：你要让用户最快记住什么
4. 视觉符号：你用什么被识别
5. 关键触点：你在哪些地方反复兑现同一件事

如果这些部分各说各话，品牌就很难形成稳定认知。

## 使用流程

1. 先确认用户的初心、服务对象和价值立场是否已经清楚。
2. 提炼一句可以承载品牌的核心表达。
3. 再把它展开到定位、命名、Slogan。
4. 再看视觉符号和关键触点怎么围绕同一价值服务。
5. 输出一版可继续打磨的话语体系草案。

阅读 [references/brand-narrative-playbook.md](references/brand-narrative-playbook.md) 获取表达层级、常见错法和输出模板。

## 什么样的输出是对的

- 核心表达主线
- 定位句草案
- 命名方向
- Slogan 方向
- 视觉符号方向
- 关键触点建议

## 使用边界

- 不要在底层命题没清楚时硬做表达。
- 不要把命名、Slogan、视觉当成彼此无关的工作。
- 如果用户还在 0 到 1 阶段、尚无任何品牌素材，这一节的输出应是方向草案，不是表达缺口诊断。
- 如果问题已经回到产品价值和用户任务，要继续路由到产品价值塑造。
