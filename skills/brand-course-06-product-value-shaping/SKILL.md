# 第五节 - 产品价值塑造

## 概览

这支 lesson skill 负责把产品价值说清楚。它不是帮用户堆功能卖点，而是帮用户回答：用户到底在什么场景下“雇佣”这个产品，他真正买走的是哪一种状态变化，以及叙事和体验是否真的对得上。

## 互动纪律

- 进入这一节，不等于马上重写完整卖点体系。
- 如果目标用户、使用场景、A/B 状态还不够清楚，先补 1-2 个验证问题。
- 用户刚确认一个产品或一个服务入口时，只能先帮助收窄 JTBD 判断，不能立刻把整套价值结构、营销结构一起讲完。
- 只有在关键场景和任务已经足够清楚时，才进入卖点与体验结构化建议。

## 角色边界

- 这支 lesson skill 负责把产品价值和 JTBD 做深，不负责重做全局系统判断。
- 它要先轻量检查：产品问题是不是其实被经营、渠道或品牌表达问题伪装出来的。
- 如果产品价值确实是主突破口，就在这一节把场景、任务、卖点和体验关系讲透。
- 如果产品只是表层症状，要明确指出并切回对应模块。

## 适用场景

- 用户觉得产品卖点很多，但说不出最核心的一句
- 用户想用 JTBD 重新梳理产品价值
- 用户在问产品到底解决了用户什么任务
- 用户担心品牌说得很好，但产品体验接不住

## 核心逻辑

用户买的不是产品本身，而是“从 A 状态到 B 状态的船票”。

这一节的核心有两层：

1. 用 JTBD 判断，用户在什么场景下雇佣这个产品
2. 用价值公式判断，产品价值 = 动人叙事 + 超预期体验

如果叙事太大、体验太弱，品牌承诺就会空掉。  
如果体验不错、叙事说不清，产品价值又很难被快速理解。

## 使用流程

1. 先明确目标用户和使用场景；如果信息不足，先补关键事实。
2. 判断用户原来的 A 状态和想去的 B 状态。
3. 提炼 JTBD：用户到底雇佣这个产品完成什么任务。
4. 如果 JTBD 还不稳定，就先停在任务判断，不急着组织完整价值结构。
5. 组织产品价值：主卖点、辅助卖点、背书、场景说明。
6. 检查对外叙事和真实体验是否一致。

阅读 [references/product-value-playbook.md](references/product-value-playbook.md) 获取 JTBD 问法、价值公式和输出模板。

## 什么样的输出是对的

- 目标用户和关键场景
- A 状态到 B 状态的变化
- JTBD 判断
- 主卖点 / 辅助卖点 / 背书 / 场景说明
- 叙事和体验的匹配情况
- 下一步最该验证什么

## 使用边界

- 不要把功能清单直接当成产品价值。
- 不要让品牌承诺高于产品真实体验。
- 如果问题更像渠道匹配或闭环问题，要继续路由到后面的课程。
- 不要因为刚确认一个产品入口，就默认后面的承接与闭环已经成立。
