# 产品价值塑造说明

## 先看用户为什么“雇佣”你

这一节不是从功能开始，而是从任务开始。

先问：

1. 用户在什么场景下会想到这个产品
2. 他原来处在什么 A 状态
3. 他希望去到什么 B 状态
4. 他为什么不直接选别的替代方案

## JTBD 的作用

JTBD 不是让你写得更学术，而是帮你看清：

- 用户到底在解决什么任务
- 这个任务发生在什么场景
- 产品的价值是效率、信任、身份、体验，还是别的变化

## 价值公式

产品价值 = 动人叙事 + 超预期体验

### 动人叙事

回答：用户为什么愿意先理解你、相信你、记住你。

### 超预期体验

回答：用户真正使用后，是否觉得这个产品比预期更好，愿意复购或转介绍。

## 卖点结构

- 主卖点：最该被记住的一件事
- 辅助卖点：补充证明为什么它成立
- 背书：为什么值得信
- 场景说明：它在什么情境里最有用

## 常见错法

- 把一串功能罗列当成价值表达。
- 只讲成分、工艺、参数，却没有说明用户状态怎么变化。
- 品牌叙事很动人，但真实体验接不住。
- 体验不错，但卖点表达太散，用户记不住。

## 适合追问的问题

- 用户最典型的使用场景是什么？
- 他用这个产品前后，状态有什么具体变化？
- 他真正雇佣你的，是解决效率问题、信任问题、身份问题还是体验问题？
- 哪个卖点最值得被放大，哪些只是辅助证明？
- 用户使用后的真实体验，有没有高于你对外说的那件事？

## 输出模板

- 目标用户
- 核心场景
- A 状态 -> B 状态
- JTBD 判断
- 主卖点 / 辅助卖点 / 背书 / 场景说明
- 叙事与体验是否一致
- 最该补的验证动作

## 什么时候继续往下走

当产品价值已经说清楚，下一步就该进入后面的课程，去判断：

- 这个产品和哪个渠道更匹配
- 应该先在哪个渠道跑出验证
- 怎么把产品、渠道、内容拧成闭环

也就是后面的 `brand-course-07-channel-product-loop` 和 `brand-course-08-minimum-marketing-loop`。
