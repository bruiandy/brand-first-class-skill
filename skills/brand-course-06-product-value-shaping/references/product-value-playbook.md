# 产品价值塑造说明

## 主原则

这一节不是在整理功能，而是在判断：

- 用户为什么在这个场景下会雇佣你
- 他原来的替代方案为什么不够
- 他为什么愿意为这个变化买单
- 这套价值有没有被真实体验接住

如果这些没有判断清楚，后面的卖点组织都会停留在表面。

## 一. 完整 JTBD 分析

JTBD 不是让你写得更学术，而是帮你看清：

- 用户到底在解决什么任务
- 这个任务发生在什么场景
- 用户原来怎么解决
- 他为什么不继续用旧方案
- 他此刻真正想完成什么进步

先问基础四问：

1. 用户是谁
2. 在什么场景下会想到这个产品
3. 他原来处在什么 A 状态
4. 他希望去到什么 B 状态

再问替代方案四问：

1. 他原来怎么解决
2. 为什么旧方案不够
3. 你真正替代掉的是什么
4. 你多出来的价值是什么

这里要注意：

- 替代方案不只是竞品，也包括用户原来“凑合着怎么解决”
- 很多时候，用户替代的不是另一款产品，而是“不做、硬扛、手动处理、用别的品类替代”

同时要提醒自己：

- 用户雇佣一个产品，不只是在买功能结果
- 很多时候，他也在买情绪上的安心感，或者社会层面的认同感
- 所以不要只停留在功能任务，也要看功能、情感、社会这三层变化

## 二. 价值翻译与卖点收束

功能、参数、工艺是证据，不是卖点。

真正的价值翻译，要回答：

- 用户为什么会在这个场景下选你
- 他到底从什么状态去到什么状态
- 他为什么愿意为这个变化付钱
- 他最终买到的是一个什么更确定的结果

如果一段表达只能说明“我们做了什么”，但不能说明“用户为什么买”，它就还不是价值表达。

也就是说：

- 功能不是价值
- 参数不是价值
- 工艺不是价值
- 它们最多只是证明这个价值成立的证据

收束规则：

- 只能有一个最该被记住的主卖点
- 辅助卖点是补充证明，不是和主卖点并列竞争
- 背书是让人更信，不是另起一套新价值
- 场景说明是让用户更快代入，不是再补一堆功能

JTBD 分析的最后目标，不是发现很多价值，而是帮助你收束出一个最该被记住的主卖点。

产品价值 = 动人叙事 + 超预期体验

### 动人叙事

回答：用户为什么愿意先理解你、相信你、记住你。

### 超预期体验

回答：用户真正使用后，是否觉得这个产品比预期更好，愿意复购或转介绍。

## 三. 叙事与体验一致性核对

- 主卖点：最该被记住的一件事
- 辅助卖点：补充证明为什么它成立
- 背书：为什么值得信
- 场景说明：它在什么情境里最有用

固定核对：

- 对外说的那件事，用户用了以后是否真的感到
- 品牌叙事和产品体验是不是同一条线
- 用户真正感知到的价值，是不是你最想放大的那一个
- 产品体验有没有高于对外承诺，而不是低于承诺

如果体验接不住，叙事越大，反而越危险。

## 四. 小闭环标准

产品价值这一节基本打通，至少要满足：

- 已经知道目标用户和最典型场景
- 已经知道 A 状态和 B 状态
- 已经知道用户真正雇佣产品完成什么任务
- 已经知道原来的替代方案为什么不够
- 已经收束出一个最该被记住的主卖点
- 已经判断这主卖点有没有被真实体验接住

如果这些还没有成立，这一节就还没通。

## 五. 常见错法

- 把一串功能罗列当成价值表达。
- 只讲成分、工艺、参数，却没有说明用户状态怎么变化。
- 品牌叙事很动人，但真实体验接不住。
- 体验不错，但卖点表达太散，用户记不住。

## 六. 适合追问的问题

- 用户最典型的使用场景是什么？
- 他用这个产品前后，状态有什么具体变化？
- 他真正雇佣你的，是在解决功能问题、情感问题，还是社会认同问题？
- 他原来为什么不继续用旧方案？
- 哪个卖点最值得被放大，哪些只是辅助证明？
- 用户使用后的真实体验，有没有高于你对外说的那件事？

## 七. 上下游检查

### 向上检查：是否承接第五节品牌叙事

- 这套卖点和场景说明，是不是在承接第五节最想让用户记住的那件事
- 这套产品价值，会不会把品牌带向它最不想被误解成的样子

### 向下检查：能不能进入第七节和第八节

- 这个主卖点，能不能进入第七节的渠道匹配判断
- 这个主卖点，能不能在第八节里被当成真正能转化的单卖点

如果上下游任一边断了，这套产品价值都还不够稳定。

## 八. 输出模板

- 目标用户
- 核心场景
- A 状态 -> B 状态
- JTBD 判断
- 替代方案判断
- 主卖点 / 辅助卖点 / 背书 / 场景说明
- 叙事与体验是否一致
- 和第五节叙事是否一致
- 能不能进入第七节和第八节
- 最该补的验证动作

## 九. 什么时候继续往下走

当产品价值已经说清楚，下一步就该进入后面的课程，去判断：

- 这个产品和哪个渠道更匹配
- 应该先在哪个渠道跑出验证
- 怎么把产品、渠道、内容拧成闭环

也就是后面的 `brand-course-07-channel-product-loop` 和 `brand-course-08-minimum-marketing-loop`。
