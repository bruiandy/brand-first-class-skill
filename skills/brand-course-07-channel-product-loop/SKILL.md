# 第六节 - 产品×渠道闭环

## 概览

这支 lesson skill 负责判断产品和渠道是否真的能拧在一起。它不是默认“先有产品，再去找渠道”，而是反过来思考：这个渠道的逻辑是什么，它需要什么样的产品规格、价格结构、内容承接和成交方式。

## 适用场景

- 用户在问这个产品更适合哪个渠道
- 用户发现线上线下都做了，但哪个都没打透
- 用户想知道不同渠道是否应该配不同规格和价格
- 用户想从渠道逻辑倒推产品组合、包装或成交方式

## 核心逻辑

渠道不是中性的分发工具，它会反过来塑造产品打法。

这一节的关键不是“多铺几个渠道”，而是先回答：

1. 哪个渠道最匹配当前产品和团队能力
2. 这个渠道要求什么样的规格、价格和表达
3. 现有产品是否需要为这个渠道重新设计

如果产品和渠道没有形成闭环，再多动作都只是分散消耗。

## 使用流程

1. 先判断当前有哪些候选渠道，以及各自的逻辑差异。
2. 再看现有产品是否匹配这些渠道的成交方式。
3. 从目标渠道倒推规格、价格、卖点和承接方式。
4. 只选一个最值得先打透的渠道方向。
5. 输出产品×渠道的适配方案和暂时不该开的口子。

阅读 [references/channel-product-playbook.md](references/channel-product-playbook.md) 获取适配判断、常见错法和输出模板。

## 什么样的输出是对的

- 当前最值得优先的渠道
- 渠道选择的原因
- 产品规格和价格需要怎么调
- 内容或承接方式需要怎么配
- 暂时不该做的渠道动作
- 进入下一节前还要验证什么

## 使用边界

- 不要默认同一款产品、同一套价格能直接套进所有渠道。
- 不要在渠道逻辑还没看清时就全面铺开。
- 如果产品价值本身还没说清楚，要先回到上一节。
- 如果渠道和产品已经基本匹配，下一步应转到最小营销闭环验证。
