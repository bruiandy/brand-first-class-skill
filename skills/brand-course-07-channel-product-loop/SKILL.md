# 第六节 - 产品×渠道闭环

## 概览

这一节对应的工具负责判断产品和渠道是否真的能拧在一起。它不是默认“先有产品，再去找渠道”，而是反过来思考：这个渠道的消费逻辑是什么，它需要什么样的产品品项、规格、价格结构、包装、内容承接和成交方式，以及团队和动销能力接不接得住。

## 协议层

这支 skill 保留原有产品与渠道闭环判断逻辑，同时遵守品牌第一课的统一工作流协议：先轻分诊，再深诊断，信息足够后输出本节专属沟通纪要。

### 协议目标

- 不把单节 skill 当成一次性答案生成器，而是当成一段可收束的诊断流程。
- 先判断用户问题是否真正命中本节；如果不命中，要指出更适合进入哪一节，不要硬答。
- 信息不足时，只输出初步判断和关键追问，不输出完整渠道方案和正式沟通纪要。
- 信息足够时，必须收束成本节专属产物，让下游 skill 可以继续读取。

### 统一工作原则

- 单节被直接调用时，第一轮必须先做 `轻分诊`：用 3-4 个判断点说明这是不是产品渠道适配问题、是否上游产品价值没清楚、还缺什么事实。
- 第一轮最多追问 1-2 个高杠杆问题，不要一次列出长问卷。
- 只要主产品、候选渠道、价格规格、团队承接或动销方式不清楚，所有判断都要标注为“初步判断”。
- 进入 `深诊断` 后，每一轮只推进一个关键判断，重点检查渠道消费逻辑、产品四维适配、团队承接和动销标准化。
- 达到收束条件后，不要继续泛聊，必须输出本节正式沟通纪要。

### 模式分层

这支 skill 默认支持 3 个 mode，不需要用户手动选择，由信息充分程度自动进入：

1. `轻分诊模式`
   - 适合用户只问某个渠道能不能做，或者线上线下都铺了但没打透。
   - 输出初步判断、是否命中本节、最需要补的渠道和产品事实。
2. `深诊断模式`
   - 适合用户愿意补信息，需要进入产品渠道适配判断。
   - 输出优先渠道、消费逻辑、产品四维适配、团队和动销要求。
3. `正式交付模式`
   - 适合信息已经足够，本节可以收束。
   - 输出正式 `渠道闭环沟通纪要`。

### 输入协议

- 优先读取 `产品价值沟通纪要`，尤其关注目标用户、关键场景、主卖点和价值证据。
- 也可以读取 `经营联动沟通纪要`，确认渠道、团队、毛利和经营导向。
- 如果没有上游纪要，也可以直接开始，但要先补齐：
  - 主产品和当前价格 / 规格 / 包装
  - 候选渠道或当前主渠道
  - 目标人群和消费场景
  - 团队承接能力
  - 当前动销方式或成交方式
- 如果用户只给一个渠道名称，不要立刻判断适不适合，先补渠道消费逻辑和产品四维信息。

### 沟通流程

1. `轻分诊`
   - 先判断这是不是产品渠道适配问题，还是产品价值、经营现实或营销闭环问题伪装成渠道问题。
   - 第一次回应只做 3-4 个关键判断点，不直接给完整渠道方案。
2. `深诊断`
   - 先判断当前更适合品牌驱动还是渠道驱动。
   - 再从渠道消费逻辑倒推产品四维适配：品项、包装、价格、规格。
   - 再判断团队承接和动销标准化是否接得住。
3. `收束输出`
   - 当产品与渠道匹配关系足够清楚时，输出 `渠道闭环沟通纪要`。
   - 输出后明确下一步通常进入 `brand-course-08-minimum-marketing-loop`，验证单渠道、单人群、单卖点的最小闭环。

### 收束条件

只有满足下面条件，才输出正式 `渠道闭环沟通纪要`：

- 已经能判断当前最值得优先的渠道
- 已经能判断更适合品牌驱动还是渠道驱动
- 已经能说清该渠道的消费逻辑
- 已经能判断产品品项 / 包装 / 价格 / 规格是否需要调整
- 已经能判断团队和动销能力是否接得住
- 已经能指出暂时不该做的渠道动作

如果这些条件还不成立，就继续追问，不要强行输出完整渠道打法。

### 正式产物

本节正式产物统一叫：

`渠道闭环沟通纪要`

正式输出必须遵守统一外壳：当前判断、关键依据、当前最重要的主线 / 矛盾 / 优先级、当前不要做什么、下一步建议、可供下一节读取的关键结论。

在这个统一外壳下，本节固定结构为：

1. 当前最值得优先的渠道
2. 当前路径：品牌驱动 / 渠道驱动
3. 渠道选择依据
4. 渠道消费逻辑
5. 产品四维适配：品项 / 包装 / 价格 / 规格
6. 内容、承接和动销方式
7. 团队能不能接得住
8. 暂时不该做的渠道动作
9. 关键依据
10. 下一节最小闭环要重点验证什么

### 下游读取协议

- `brand-course-08-minimum-marketing-loop` 读取本纪要里的 `优先渠道`、`目标人群`、`主卖点`、`产品四维适配` 和 `动销方式`，继续验证最小盈利闭环。
- 如果本纪要显示产品价值不清，回到 `brand-course-06-product-value-shaping`，不要直接进入闭环验证。

### 衔接规则

- 如果当前问题其实属于第六节产品价值问题，要明确提示先回第六节补主卖点和产品价值，不在本节强行做渠道方案。
- 如果本节已经判断完成，要明确提示可以进入第七节最小营销闭环，验证单渠道、单人群、单卖点是否跑得通。
- 不允许一口气把内容矩阵、全渠道放大和增长 SOP 都讲完；本节只解决“产品和渠道是否形成闭环”问题。

## 互动纪律

- 进入这一节，不等于马上画出完整渠道方案。
- 如果当前渠道、产品规格、价格结构、团队承接方式还不清楚，先补 1-2 个验证问题。
- 用户刚确认一个渠道或一个产品时，只能先帮助判断匹配度，不能立刻把整套打法和后续营销闭环都讲完。
- 只有在产品与渠道的基础匹配、团队承接和动销方式都已经基本清楚时，才进入更具体的适配方案。

## 角色边界

- 这一节对应的工具负责把产品与渠道的匹配关系做深，不负责重做总控层的系统诊断。
- 它要先轻量检查：渠道问题是不是因为产品价值不清、经营现实错误或闭环尚未成立。
- 如果渠道匹配确实是主突破口，就在这一节深挖渠道消费逻辑、产品四维适配、团队承接和动销标准化。
- 如果渠道问题只是下游表现，要明确指出并切回对应模块。

## 适用场景

- 用户在问这个产品更适合哪个渠道
- 用户发现线上线下都做了，但哪个都没打透
- 用户想知道不同渠道是否应该配不同规格和价格
- 用户想从渠道逻辑倒推产品组合、包装或成交方式
- 用户想判断现在更该走品牌驱动还是渠道驱动

## 核心逻辑

渠道不是中性的分发工具，它会反过来塑造产品打法。

这一节的关键不是“多铺几个渠道”，而是先回答：

1. 你当前更适合品牌驱动还是渠道驱动
2. 哪个渠道最匹配当前产品和团队能力
3. 这个渠道背后的消费逻辑是什么
4. 这个渠道要求什么样的品项、包装、价格和规格
5. 现有团队和动销能力接不接得住

如果产品和渠道没有形成闭环，再多动作都只是分散消耗。

## 使用流程

1. 先判断当前更适合品牌驱动还是渠道驱动；如果用户还是初创阶段、资源有限，默认优先检查渠道驱动路径。
2. 再判断当前有哪些候选渠道，以及各自的消费逻辑差异；不清楚就先补事实。
3. 再看现有产品是否匹配这些渠道的成交方式和决策节奏。
4. 从目标渠道倒推产品四维适配：品项、包装、价格、规格。
5. 再判断团队承接能力：有没有能力把这个渠道真正打透，而不是只把货铺进去。
6. 再判断动销方式是否能标准化：不同级别渠道需要什么样的活动、物料、节奏和人效配置。
7. 如果匹配关系还在早期判断阶段，就先指出最可能的问题点和待验证点。
8. 只选一个最值得先打透的渠道方向。
9. 输出产品×渠道的适配方案、团队与动销要求，以及暂时不该开的口子。

阅读 [references/channel-product-playbook.md](references/channel-product-playbook.md) 获取渠道逻辑、产品四维适配、团队承接和动销标准化判断。

## 什么样的输出是对的

- 当前最值得优先的渠道
- 当前更适合的路径：品牌驱动或渠道驱动
- 渠道选择的原因
- 这个渠道的消费逻辑
- 产品品项 / 包装 / 价格 / 规格需要怎么调
- 内容、承接和动销方式需要怎么配
- 团队能不能接得住
- 暂时不该做的渠道动作
- 进入下一节前还要验证什么

## 输出纪律

- 不要把这一节做成渠道罗列器，要先判断哪个渠道是主战场。
- 不要只讲流量大小，要讲清这个渠道背后的消费逻辑。
- 不要只讲产品适配，不讲团队承接和动销能力。
- 不要默认同一套产品四维配置能直接复制到所有渠道。
- 不要因为刚找到一个合适渠道，就自动展开多渠道联动方案。

## 使用边界

- 不要默认同一款产品、同一套价格能直接套进所有渠道。
- 不要在渠道逻辑还没看清时就全面铺开。
- 不要只看能不能进渠道，不看进去之后能不能动销。
- 如果产品价值本身还没说清楚，要先回到上一节。
- 如果渠道和产品已经基本匹配，下一步应转到最小营销闭环验证。
- 不要因为刚确认一个优先渠道，就把完整的增长打法一次性补齐。
