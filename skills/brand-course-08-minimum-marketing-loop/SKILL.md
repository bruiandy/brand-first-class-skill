# 第七节 - 构建最小营销闭环

## 概览

这支 lesson skill 负责把增长问题收束到一个最小、可验证、可复制的闭环上。它不是帮用户追求一开始的最大声量，而是帮用户先跑通第一个可盈利循环，再决定要不要标准化和放大。

## 互动纪律

- 进入这一节，不等于马上给完整营销方案。
- 如果渠道、人群、卖点、转化链路还没确认，先补 1-2 个验证问题。
- 用户刚确认一个入口或一个主产品时，只能先帮助收窄闭环判断，不能立刻把整套内容策略和放大路径全部讲完。
- 只有在闭环四要素已经足够清楚，或确实是单点闭环问题时，才进入更具体的验证动作。

## 角色边界

- 这支 lesson skill 负责把最小营销闭环做深，不负责重做整套系统诊断。
- 它要先轻量检查：闭环问题是不是因为上游的产品价值、渠道匹配或经营现实没站稳。
- 如果闭环确实是主突破口，就在这一节深挖渠道、人群、卖点和验证动作。
- 如果闭环只是最后一环的表现，要明确指出并切回对应模块。

## 适用场景

- 用户觉得不是没流量，而是没有跑通闭环
- 用户在问第一条增长路径应该怎么验证
- 用户什么渠道都试了，但没有一个形成稳定成交
- 用户想知道该怎么做单点突破，而不是继续全渠道铺开

## 核心逻辑

最小营销闭环的重点不是做得多，而是先做成一条可重复的路。

这一节先看四件事：

1. 单渠道
2. 单人群
3. 单卖点
4. 可盈利循环

如果这四件事没有同时成立，增长就很难被稳定复制。

## 使用流程

1. 先选一个最值得验证的渠道；如果信息不足，先补关键事实。
2. 再聚焦一类最明确的人群。
3. 再确定一个最有力的卖点。
4. 检查从获客、转化到复购 / 转介绍的链路是否成立。
5. 如果闭环判断还在早期，只先指出最薄弱的一环和待验证点。
6. 只有在闭环成立后，才考虑标准化和放大。

阅读 [references/minimum-marketing-loop-playbook.md](references/minimum-marketing-loop-playbook.md) 获取闭环定义、常见错法和输出模板。

## 什么样的输出是对的

- 当前最该验证的渠道 / 人群 / 卖点组合
- 这条闭环为什么最值得先跑
- 闭环链路里最薄弱的一环
- 最小验证动作
- 成立后该怎么标准化
- 暂时不要做什么

## 使用边界

- 不要在没跑通一条闭环前就追求全渠道扩张。
- 不要把内容数量误当成增长验证。
- 不要把品牌声量当成闭环成立的证据。
- 如果产品和渠道的匹配关系还没看清，要先回到上一节。
- 不要因为刚确认一个渠道或主产品，就直接输出完整内容计划或放大方案。
