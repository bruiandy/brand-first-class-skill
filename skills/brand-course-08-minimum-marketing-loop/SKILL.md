# 第七节 - 构建最小营销闭环

## 概览

这支课程工具负责把增长问题收束到一个最小、可验证、可复制、可盈利的闭环上。它不是帮用户一开始就追求最大声量，而是帮用户先用最低成本、最快速度跑通第一条盈利路径，再决定要不要承诺资源、标准化和放大。

## 协议层

这支 skill 保留原有最小营销闭环、三阶段和强信号判断逻辑，同时遵守品牌第一课的统一工作流协议：先轻分诊，再深诊断，信息足够后输出本节专属沟通纪要。

### 协议目标

- 不把单节 skill 当成一次性答案生成器，而是当成一段可收束的诊断流程。
- 先判断用户问题是否真正命中本节；如果不命中，要指出更适合进入哪一节，不要硬答。
- 信息不足时，只输出初步判断和关键追问，不输出完整营销方案和正式沟通纪要。
- 信息足够时，必须收束成本节专属产物，让用户知道第一条最小盈利路径怎么验证。

### 统一工作原则

- 单节被直接调用时，第一轮必须先做 `轻分诊`：用 3-4 个判断点说明这是不是最小闭环问题、是否上游产品或渠道没清楚、还缺什么事实。
- 第一轮最多追问 1-2 个高杠杆问题，不要一次列出长问卷。
- 只要主渠道、核心人群、主卖点、成交链路或强信号不清楚，所有判断都要标注为“初步判断”。
- 进入 `深诊断` 后，每一轮只推进一个关键判断，重点检查单渠道、单人群、单卖点、真实成交、获客成本和可重复性。
- 达到收束条件后，不要继续泛聊，必须输出本节正式沟通纪要。

### 模式分层

这支 skill 默认支持 3 个 mode，不需要用户手动选择，由信息充分程度自动进入：

1. `轻分诊模式`
   - 适合用户只说流量很多但没成交，或者想知道闭环卡在哪。
   - 输出初步判断、是否命中本节、最需要补的闭环事实。
2. `深诊断模式`
   - 适合用户愿意补信息，需要进入最小闭环验证判断。
   - 输出当前阶段、渠道 / 人群 / 卖点组合、最薄弱环节和强信号缺口。
3. `正式交付模式`
   - 适合信息已经足够，本节可以收束。
   - 输出正式 `最小闭环方案沟通纪要`。

### 输入协议

- 优先读取 `渠道闭环沟通纪要`，尤其关注优先渠道、目标人群、产品四维适配、动销方式和下一节要验证什么。
- 也可以读取 `产品价值沟通纪要`，确认最小闭环里的单卖点是否和第六节主卖点一致。
- 如果没有上游纪要，也可以直接开始，但要先补齐：
  - 当前最想验证的渠道
  - 核心人群
  - 主卖点
  - 当前内容或触达方式
  - 成交链路
  - 已经出现的成交、成本、复购或转介绍信号
- 如果用户只给播放、曝光、点赞数据，不要判断闭环成立，先补成交、成本和重复性。

### 沟通流程

1. `轻分诊`
   - 先判断这是不是最小营销闭环问题，还是产品价值、渠道适配或经营现实问题伪装成增长问题。
   - 第一次回应只做 3-4 个关键判断点，不直接给完整营销方案。
2. `深诊断`
   - 先判断当前处于验证期、承诺期还是规模化阶段。
   - 再收束单渠道、单人群、单卖点组合。
   - 再检查触达、成交、复购 / 转介绍链路和强信号。
3. `收束输出`
   - 当最小闭环结构和验证动作足够清楚时，输出 `最小闭环方案沟通纪要`。
   - 输出后明确当前应该继续验证、承诺资源、标准化，还是暂时回到上游重修。

### 收束条件

只有满足下面条件，才输出正式 `最小闭环方案沟通纪要`：

- 已经能判断当前处于验证期、承诺期还是规模化阶段
- 已经能收束出一个渠道 / 人群 / 卖点组合
- 已经能判断这条闭环为什么最值得先跑
- 已经能指出闭环链路里最薄弱的一环
- 已经能判断已经出现什么强信号、还缺什么信号
- 已经能明确最小验证动作和暂时不要做什么

如果这些条件还不成立，就继续追问，不要强行输出完整营销计划。

### 正式产物

本节正式产物统一叫：

`最小闭环方案沟通纪要`

正式输出必须遵守统一外壳：当前判断、关键依据、当前最重要的主线 / 矛盾 / 优先级、当前不要做什么、下一步建议、可供下一节读取的关键结论。

在这个统一外壳下，本节固定结构为：

1. 当前阶段：验证期 / 承诺期 / 规模化
2. 最小闭环组合：渠道 / 人群 / 卖点
3. 为什么这条闭环最值得先跑
4. 内容类型：搜索驱动 / 发现驱动 / 信任驱动
5. 闭环链路里最薄弱的一环
6. 已经出现的强信号
7. 还缺什么信号
8. 最小验证动作
9. 当前不要做什么
10. 成立后如何进入承诺资源、SOP 和放大

### 下游读取协议

- 这是八节课的阶段性收束产物。后续复盘、网站工作台或咨询陪跑，可以读取本纪要里的 `最小闭环组合`、`强信号`、`最薄弱一环` 和 `下一步动作`。
- 如果本纪要显示单卖点和第六节主卖点不一致，回到 `brand-course-06-product-value-shaping`；如果渠道和产品不匹配，回到 `brand-course-07-channel-product-loop`。

### 衔接规则

- 如果当前问题其实属于第六节产品渠道适配问题，要明确提示先回第六节补渠道闭环，不在本节强行做增长方案。
- 如果本节已经判断完成，要明确提示当前是继续验证、承诺资源、标准化，还是回到上游重修。
- 不允许一口气展开全渠道、矩阵化、资产化和长期品牌打法；本节只解决“第一条最小盈利路径是否跑得通”问题。

## 互动纪律

- 进入这一节，不等于马上给完整营销方案，更不等于直接展开多渠道动作。
- 如果主渠道、核心人群、主卖点、成交链路还没确认，先补 1-2 个验证问题。
- 用户刚确认一个入口或一个主产品时，只能先帮助收窄最小闭环，不能立刻把整套内容策略、活动节奏和放大路径全部讲完。
- 只有在闭环结构已经足够清楚，或确实是单点闭环问题时，才进入更具体的验证动作。
- 输出必须优先收束到一条最小盈利路径，而不是给一份看起来很全面的营销清单。

## 角色边界

- 这支课程工具负责把最小营销闭环做深，不负责重做整套系统诊断。
- 它要先轻量检查：闭环问题是不是因为上游的产品价值、渠道匹配或经营现实没站稳。
- 如果闭环确实是主突破口，就在这一节深挖渠道、人群、卖点、内容形态和验证动作。
- 如果闭环只是最后一环的表现，要明确指出并切回对应模块。

## 适用场景

- 用户觉得不是没流量，而是没有跑通最小盈利闭环
- 用户在问第一条增长路径应该怎么验证
- 用户什么渠道都试了，但没有一个形成稳定成交
- 用户想知道该怎么做单点突破，而不是继续全渠道铺开
- 用户想判断现在处在验证期、承诺期还是规模化阶段

## 核心逻辑

最小营销闭环的重点不是做得多，而是先做成一条可重复、可盈利的路。

这一节先看六件事：

1. 单渠道
2. 单人群
3. 单卖点
4. 真实成交
5. 获客成本可控
6. 可被重复验证

如果这六件事没有基本同时成立，增长就很难被稳定复制。

这一节还要先完成三种思维切换：

1. 从品牌广告思维，切到闭环成交思维
2. 从多渠道铺开，切到单点打透思维
3. 从内容数量思维，切到闭环效率思维

## 使用流程

1. 先判断当前是验证期、承诺期还是规模化阶段。
2. 再选一个最值得验证的渠道；如果信息不足，先补关键事实。
3. 再聚焦一类最明确的人群和一个最有力的主卖点。
4. 判断内容更接近搜索驱动、发现驱动还是信任驱动。
5. 用聚焦化、场景化、具体化、真实化去检查内容和转化动作是否在为闭环服务。
6. 检查从触达、成交到复购 / 转介绍的链路是否成立，是否已经出现强信号。
7. 如果闭环判断还在早期，只先指出最薄弱的一环和待验证点。
8. 只有在闭环成立后，才进入承诺资源、标准化和放大。

阅读 [references/minimum-marketing-loop-playbook.md](references/minimum-marketing-loop-playbook.md) 获取最小盈利闭环定义、内容四原则、三阶段方法、强信号判断和输出模板。

## 什么样的输出是对的

- 当前最该验证的渠道 / 人群 / 卖点组合
- 当前所处阶段：验证期 / 承诺期 / 规模化
- 这条闭环为什么最值得先跑
- 闭环链路里最薄弱的一环
- 当前内容类型更适合搜索驱动、发现驱动还是信任驱动
- 当前已经出现了什么强信号，还缺什么信号
- 最小验证动作
- 成立后该怎么承诺资源、做 SOP 和放大
- 暂时不要做什么

## 使用边界

- 不要在没跑通一条闭环前就追求全渠道扩张。
- 不要把内容数量误当成增长验证。
- 不要把品牌声量当成闭环成立的证据。
- 如果产品和渠道的匹配关系还没看清，要先回到上一节。
- 如果这一节押的单卖点和第六节收束出的主卖点不是一回事，要先回到上一节重新对齐。
- 不要因为刚确认一个渠道或主产品，就直接输出完整内容计划或放大方案。
- 不要把点赞、播放、曝光当成闭环成立的核心证据；先看成交、成本和重复性。
- 不要在验证期就讨论矩阵化和资产化动作。
