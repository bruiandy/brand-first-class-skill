# 品牌方法总控

## 概览

这是整个课程系统的入口。它不负责把所有问题都自己答完，而是负责接住问题、判断意图、选择路由，并让整套课程始终沿着同一条判断主线运行。

## 适用场景

当用户出现以下情况时，使用这支 skill：

- 不知道应该用哪一节课
- 想先看整套课程的总览
- 带来了一个真实案例，但问题还没整理清楚
- 想用同一套方法做诊断、拆案例、筛选或内容演绎
- 想快速理解这 8 节课之间的关系

## 这支 Skill 做什么

1. 判断用户的问题到底属于哪一类。
2. 路由到最合适的 lesson skill 或支持型 skill。
3. 如果对应 lesson skill 还没建好，就用共享方法主线先接住，并说明它属于哪一个课程槽位。
4. 保证整套系统的语言和判断顺序一致。

阅读 [references/lesson-map.md](references/lesson-map.md) 获取路由表和兜底规则。

## 路由流程

### 第一步：识别意图

先判断，用户真正要做的是什么？

- 理解整套方法
- 学某一节课
- 诊断真实品牌案例
- 拆一个公开案例或内容选题
- 做合作线索筛选
- 拿到接下来更可执行的方向

### 第二步：选择路由

使用下面的路由表。

### 第三步：回答或交接

- 如果已有对应 skill，就明确交给它。
- 如果还没有，就用共享主线先回答，并指出它属于哪一节课。
- 如果用户的问题过于模糊，只问一个聚焦问题，不要连续追问。

## 共享方法主线

所有路由都沿用同一条主线：

- 盘资源
- 经营
- 聚焦优势
- 品牌产品
- 营销闭环

`聚焦优势` 是桥梁判断，不是一个孤立口号。它通常发生在资源扫描之后、品牌与产品判断之前。

## 默认回答结构

总控回答时，保持简洁：

- 用户现在在问什么
- 最适合去哪个 skill / lesson slot
- 为什么这样路由
- 如果还没法路由，先补哪一个关键信息

## 使用边界

- 不要把所有问题都硬路由成诊断。
- 如果清晰交接更好，就不要在总控里讲太多。
- 不要把总控写成第二份课程正文。
- 如果问题模糊，最多问一个聚焦问题再路由。
- 如果用户要的是整节课，就指向对应 lesson slot，而不是每次都把整套课重讲一遍。
